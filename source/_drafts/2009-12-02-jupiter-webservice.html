I was thinking that we could use Jupiter's eclipse front end to talk to a Ruby webservice.

Setting up a review:
1. Configure the review using the jupiter UI.
2. Select members to participate in the review.
3. Publish review information to the webservice.

Loading a review:
1. Register yourself with the web service if you haven't done so.
  - username, email, picture?
2. Webservice will publish to you the reviews that you are part of.
3. Conduct review.
4. Publish results.

Going through the review:
1. Pull changes from the webservice.
2. Go through the moderated code review.
3. Publish changes.

The author can then edit the changes and publish information back to the web service.

Work to be done:

1. Webservice
  Webservice configuration.  URL, admin password, etc.  No web interface for the beginning stages.
2. UI Updates.



At the CSDL (link it) we used an in-house product called Jupiter.  My friend Julie(link her blog) provides the maintenance of the tool with each new release of Eclipse.  She does a great job.  Jupiter's code review functionality can't be beat.  With Jupiter you can review code in your Eclipse IDE.  You get to use all of the IDE functionality that you are used to with some special Jupiter review sauce.  The huge draw back of Jupiter is that it takes a really long time to setup the review because you have to create the review in eclipse and then send out an email to your developers saying what to review.  Then you need to commit a .jupiter file, which is a pain in the butt.  You also have to make sure that the .jupiter file resides in a branch because you don't want that file to live in your trunk source.  Then you have to make sure you do a code freeze because you don't want people reviewing a different change set.  You could specify which revisions to review, but then the developers have to switch to the correct revision.  As you start to get lots of developers commiting their changes you'll notice that the review process of Jupiter just doesn't scale.

(Talk to Julie about improving the review process?  A Jupiter server.  Webapp that allows you to link up reviews and see what is going on?)
